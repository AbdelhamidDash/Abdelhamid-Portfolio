NetWatcher

NetWatcher is a lightweight Android library for real-time network connectivity monitoring. It wraps Android's ConnectivityManager into a reactive, lifecycle-aware API — giving apps instant awareness of connectivity changes without leaking memory or registering duplicate receivers.

Available on GitHub

Introduction

Handling network state in Android is error-prone. The legacy BroadcastReceiver approach leaks if not unregistered, and the modern NetworkCallback API requires careful lifecycle management that many developers get wrong.

NetWatcher encapsulates all of this behind a single LiveData and Flow stream. Observe it from any ViewModel or Composable and the library automatically registers and unregisters the underlying callback based on the lifecycle — zero leaks guaranteed.

Contributions

  • Built the core NetworkCallback wrapper with automatic lifecycle registration and cleanup.
  • Exposed connectivity state as both LiveData and Kotlin Flow for flexibility across MVVM and Compose architectures.
  • Added network type detection (WiFi, cellular, ethernet) so apps can react differently per connection type.
  • Wrote unit tests covering connectivity state transitions with a faked ConnectivityManager.
  • Published to JitPack with zero-dependency design — the library adds no transitive dependencies to the host app.

Key Highlights

Zero Leaks

Lifecycle-aware registration ensures callbacks are always cleaned up automatically.

Reactive API

Observe connectivity as LiveData or Kotlin Flow — fits any modern Android architecture.

Network Type Aware

Distinguishes between WiFi, cellular, and ethernet so apps can adapt behavior per type.

Zero Dependencies

Adds nothing to your dependency tree — pure Android SDK usage only.
