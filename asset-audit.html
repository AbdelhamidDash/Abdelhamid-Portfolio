Asset Audit

Asset Audit System is a Flutter app designed for large enterprises managing distributed inventory across hundreds of locations. It enables branch leaders to systematically scan assets, automatically detect duplicates and data quality issues, and maintain a unified inventory database — all while working offline in the field.

Introduction

Large enterprises managing 600+ distributed branches face a critical gap: no unified system to track, validate, and reconcile physical assets across locations. Duplicate entries, manual spreadsheets, and disconnected workflows lead to financial inaccuracies that take weeks — sometimes months — to untangle.

Asset Audit brings systematic scanning, intelligent deduplication, and real-time data quality assurance to mobile. Built for field teams at IDH Corp in Egypt, the app lets branch leaders audit inventory on the spot, flag duplicates automatically, and submit validated records — all without an internet connection.

The app is built with Flutter using an offline-first architecture. Local caching ensures full functionality in the field, while a smart sync engine handles conflict resolution when connectivity is restored. Fuzzy matching (Levenshtein distance) powers the duplicate detection engine, catching near-identical serial numbers and asset descriptions across branches.

Contributions

  • Integrated barcode scanning via mobile_scanner, enabling real-time camera-based asset lookup and inventory entry without manual input.
  • Built a duplicate detection engine using fuzzy matching (Levenshtein distance) to cross-check asset serial numbers and descriptions across all branches, surfacing likely duplicates before they corrupt the dataset.
  • Implemented branch authentication & role-based access so branch leaders see only their location's data while admins retain cross-branch visibility.
  • Created a data quality validation layer that checks required fields, format constraints, and logical consistency before any record is committed or synced.
  • Designed an offline-first sync layer with Hive local caching so the app functions fully without connectivity, then intelligently merges changes and resolves conflicts on reconnect.
  • Built an audit trail logging system that records every scan, edit, and sync event with timestamps and user identity for full traceability.
  • Designed and implemented the entire field-facing UI/UX in Flutter — scan flows, duplicate review screens, and sync status indicators — optimised for non-technical branch leaders.
